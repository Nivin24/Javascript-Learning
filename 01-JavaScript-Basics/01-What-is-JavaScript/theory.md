# What is JavaScript?

## Introduction

JavaScript is a **high-level, interpreted programming language** that is one of the core technologies of the World Wide Web, alongside HTML and CSS. While HTML provides structure and CSS provides styling, JavaScript brings **interactivity** and **dynamic behavior** to web pages.

---

## Brief History

- **1995**: Created by Brendan Eich at Netscape in just 10 days
- **1997**: Standardized as ECMAScript (ES) by ECMA International
- **2015**: ES6 (ES2015) introduced major features like arrow functions, classes, modules
- **Today**: JavaScript runs everywhere—browsers, servers (Node.js), mobile apps, IoT devices

---

## Where Does JavaScript Run?

### 1. **Client-Side (Browser)**
JavaScript runs in web browsers to:
- Respond to user actions (clicks, typing, scrolling)
- Validate forms before submission
- Create animations and visual effects
- Fetch data from servers without reloading the page (AJAX)

**Examples**: Interactive forms, image sliders, real-time chat applications

### 2. **Server-Side (Node.js)**
With Node.js, JavaScript can run on servers to:
- Handle HTTP requests
- Access databases
- Process files
- Build REST APIs

**Examples**: Express.js backends, real-time applications with Socket.io

### 3. **Other Platforms**
- **Mobile apps**: React Native, Ionic
- **Desktop apps**: Electron (VS Code, Slack, Discord)
- **IoT devices**: Johnny-Five framework

---

## JavaScript vs ECMAScript

- **JavaScript**: The language implementation (what you write)
- **ECMAScript**: The specification/standard (the rules)

Think of ECMAScript as the "blueprint" and JavaScript as the "building" constructed from it.

**Versions you'll hear about:**
- ES5 (2009) – Widely supported, older syntax
- ES6/ES2015 (2015) – Major update with modern features
- ES2016, ES2017... – Yearly updates with incremental improvements

---

## What Makes JavaScript Special?

1. **Interpreted Language**: No compilation needed; runs directly in browsers
2. **Dynamically Typed**: Variables can hold any type of data
3. **Event-Driven**: Responds to user interactions and system events
4. **Asynchronous**: Can handle multiple operations simultaneously (callbacks, promises, async/await)
5. **Prototype-Based**: Object-oriented programming through prototypes (not classes, though ES6 added class syntax)

---

## What Can You Build with JavaScript?

- **Websites**: Interactive user interfaces (React, Vue, Angular)
- **Web Applications**: Gmail, Google Maps, Trello
- **Mobile Apps**: Cross-platform apps with React Native
- **Desktop Apps**: Electron-based applications
- **Games**: Browser-based games with Canvas API or libraries like Phaser
- **Server Applications**: APIs and backends with Node.js
- **Machine Learning**: TensorFlow.js for browser-based ML

---

## JavaScript in the Browser

When you load a web page:
1. Browser downloads HTML, CSS, and JavaScript files
2. HTML parser builds the DOM (Document Object Model)
3. JavaScript engine (V8 in Chrome, SpiderMonkey in Firefox) executes JS code
4. JavaScript can manipulate the DOM, change styles, handle events

---

## Key Takeaways

✅ JavaScript makes web pages interactive and dynamic  
✅ It runs in browsers (client-side) and servers (Node.js)  
✅ ECMAScript is the standard; JavaScript is the implementation  
✅ Modern JavaScript (ES6+) has powerful features for cleaner code  
✅ You can build almost anything with JavaScript today
